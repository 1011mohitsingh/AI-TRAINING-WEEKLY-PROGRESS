Code walkthrough
Step 1 — Python list
n = 1_000_000   A = [30] * n B = [30] * n result = [0] * n   for i in range(n): result[i] = A[i] + B[i] # ↑ visits every element one by one # type check + object overhead × n

Why slow: Python loops are interpreted line by line. Each iteration has overhead — type checking, memory allocation, object handling — repeated n times.

Step 2 — NumPy array
import numpy as np   a = np.arange(n) b = np.arange(n)   c = a + b # ← single vectorized call   # No loop. No per-element overhead. # Runs compiled C code under the hood.

Why fast: NumPy stores data in contiguous memory of a single type. The + calls pre-compiled C/SIMD code that processes the entire array at once — no Python interpreter per step.

Step 3 — measuring the difference
import time   start = time.time() for i in range(n): result[i]=A[i]+B[i] list_time = time.time() - start   start = time.time() c = a + b np_time = time.time() - start   speedup = list_time / np_time print(round(speedup, 2)) # run to see →
Python list — stores Python objects, each with type info & ref count. Loop overhead compounds with n. NumPy array — raw C memory block, fixed dtype, vectorized CPU instructions (SIMD). One call, no overhead.
